## 아이디어 #1 — 버려지는 재활용 자원 포획 시스템

**미션**: 생활환경 (자원순환)
**핵심 키워드**: 재활용 원료 가치화, AI 발생 예측, 자원 포획, 처리 비용 상쇄

---

### 한 줄 요약

> 처리 비용이 3배 오른 지금, 일반쓰레기로 버려지는 재활용 자원을 AI로 찾아내 수거·판매해 처리 비용을 자체 충당하는 선순환 구조를 설계한다.

---

### 문제 정의

수도권 직매립 금지 시행 이후 민간 소각 위탁 단가가 최고 **259,500원/톤**까지 치솟았다. 지자체들은 비용 폭등을 고스란히 예산으로 떠안고 있다.

그런데 이 상황을 다르게 보면:

> 처리 비용이 3배 올랐다 = 처리 안 해도 되는 재활용 자원의 가치가 3배 올랐다

지금 한국 생활폐기물 안에는 **분리수거를 안 해서 그냥 태워지는** 플라스틱, 종이, 금속, 유리가 섞여 있다. 이것들은 재활용 시장에서 원료로 팔린다. 버려지고 있는 돈이다.

문제는 **어디서, 어떤 건물에서, 얼마나** 버려지는지를 아무도 정확히 모른다는 것.

---

### 핵심 아이디어

**AI로 "재활용 자원이 쓰레기로 버려지는 핫스팟"을 찾아내고, 거기에 집중 수거를 배치해 원료로 판매한다.**

```
[1단계] 핫스팟 예측
지역별 인구 구성 + 건물 유형 + 과거 재활용률 + 폐기물 발생량 데이터
→ AI 모델: "어느 동네, 어느 건물 유형에서 재활용 가능 자원이 
  일반쓰레기로 가장 많이 버려지는가" 예측

[2단계] 집중 수거
핫스팟 지역에 선별 수거 자원 집중 배치
일반 수거와 분리해서 고품질 재활용 원료 확보

[3단계] 원료 판매
자원순환정보시스템 시장 가격 기반으로 재활용 업체에 판매
수익 → 지자체 소각 처리 비용 일부 상쇄

[4단계] 피드백 루프
수거 결과 데이터 → 모델 재학습 → 핫스팟 정밀도 향상
```

---

### 왜 지금인가

- 처리 비용 폭등으로 **재활용 포획의 경제성이 처음으로 확실히 플러스**가 됐다
- 기존에는 선별 수거 비용 > 원료 판매 수익 → 유인 없었음
- 지금은 소각 위탁 단가 급등 → 선별 수거로 소각량 줄이면 절감 + 판매 수익 이중 이득

---

### 차별점 (기존 아이디어와 비교)

| 기존 접근 | 이 아이디어 |
|---------|-----------|
| 재활용률을 높이자 (시민 계도) | 어디서 못 하는지 AI로 찾아서 자원 투입 |
| 처리 시설을 늘리자 | 처리할 양 자체를 수익원으로 전환 |
| 비용을 누가 낼지 정하자 | 비용을 수익으로 바꾸는 구조 설계 |
| 규제·의무화 | 경제적 인센티브 기반 자발적 참여 |

---

### 활용 데이터

| 데이터 | 기관 | 용도 |
|--------|------|-----|
| 지자체별 폐기물 발생·처리 현황 | 환경부 환경통계포털 | 발생량 기반 예측 변수 |
| 재활용 가능 자원 시장 가격 | 자원순환정보시스템 | 원료 판매 수익 산정 |
| 재활용 선별 현황 | 환경부 | 지역별 현재 선별률 |
| 인구·주택 유형 통계 | 통계청 | 핫스팟 예측 변수 |
| 생활폐기물 성상 분석 자료 | 환경부 | 폐기물 내 재활용 가능 비율 |

---

### 정책 제언 방향

1. **AI 기반 자원 핫스팟 지도** 표준화 → 전국 지자체 공유 플랫폼
2. 핫스팟 지역 집중 수거 시범 사업 → 비용 절감 실증
3. 수거 수익의 일부를 해당 지역 주민 환경 포인트로 환류 (참여 유인)
4. 장기적으로 소각 위탁 물량 자체를 줄여 민간 단가 협상력 회복

---

### 한계 및 보완 필요 사항

- 선별 수거 품질이 낮으면 원료 가치 하락 → 수거 방식 설계 중요
- 재활용 시장 가격 변동성 → 수익 안정성 보완 필요
- 시범 지역 선정 및 초기 투자 비용 재원 확보 방안 필요

---

### 심사 기준 대응

| 심사 항목 | 대응 |
|---------|-----|
| 데이터 융합성 | 폐기물 발생 + 시장 가격 + 인구·건물 데이터 교차 분석 |
| 창의성 | "처리 비용 문제"를 "수익 창출 기회"로 프레임 전환 |
| 실현 가능성 | 기존 수거 인프라 활용, 법 개정 불필요 |
| 사회적 파급력 | 지자체 재정 압박 해소 + 재활용률 향상 + 탄소 감축 |
| 논리적 타당성 | 처리 비용 급등 = 재활용 경제성 급등 이라는 명확한 수치 근거 |

## 아이디어 #2 — 내 대기질 행동 코치

**미션**: 대기환경 (PM2.5·오존)
**핵심 키워드**: 개인화 위험 알림, 건강 프로필, 흡입량 기반 위험도, 행동 가이드

---

### 한 줄 요약

> 에어코리아는 "나쁨"이라는 숫자만 주고, 내가 오늘 뭘 해야 하는지는 아무도 안 알려준다. 건강 프로필을 등록해두면 내 몸 기준으로 위험한 순간마다 먼저 알려주는 서비스.

---

### 문제 정의

에어코리아는 하루 4회 예보 + 지역 평균 색깔(좋음/나쁨)을 제공한다. 취약계층 구분도 유치원·학교·노인시설 등 **기관 단위**이지, 개인 단위가 아니다.

천식 환자와 건강한 30대가 같은 "나쁨" 정보를 받는다. 오늘 달리기할 사람과 실내에만 있을 사람이 같은 정보를 받는다. 개인에게 필요한 정보는 따로 있다.

---

### 핵심 아이디어

**건강 프로필 1회 등록 → 내 몸 기준 위험 시 자동 알림**

```
[최초 1회 설정]
건강 상태: 천식 / COPD / 심장질환 / 임신 / 어린이 / 노인 / 일반
주요 야외 활동: 조깅 / 자전거 / 등산 / 배달 / 농업 / 통학
주요 행동 시간: 출퇴근 시간, 운동 시간 등록

        ↓ 평상시에는 아무 알림 없음

[위험 감지 시 자동 Push]
에어코리아 실시간 + CMAQ 시간대별 예보 + 기상 데이터 융합
→ 내 건강 프로필 × 등록 활동 시간대 × 예상 농도 대입
→ 흡입량 기반 위험 스코어 계산
→ 위험 수준 초과 시 먼저 알림 발송

예시 알림:
"오늘 오후 6시 조깅 계획 기준 위험합니다.
 PM2.5 58μg/m³ 예상 → 천식 환자 30분 달리기 흡입량 과다.
 ▶ 내일 오전 7시로 변경 시 안전 (예상 19μg/m³)"
```

---

### 흡입량 계산 모델 (기존 앱에 없는 것)

| 활동 | 분당 호흡량 |
|------|----------|
| 달리기 | 40L/분 |
| 자전거 | 25L/분 |
| 빠른 걷기 | 18L/분 |
| 일반 걷기 | 12L/분 |
| 앉아있기 | 8L/분 |

`PM2.5 농도 × 활동 시간 × 분당 호흡량 × 건강 민감도 계수 = 개인화 위험 스코어`

숫자는 모두 공공데이터에 있다. 아무도 "사람의 행동"에 연결하지 않았을 뿐.

---

### 활용 데이터

| 데이터 | 기관 | 가점 |
|--------|------|------|
| 에어코리아 실시간 측정 API | 환경공단 | **가점** |

---

### 심사 기준 대응

| 심사 항목 | 대응 |
|---------|-----|
| 데이터 융합성 | 에어코리아 실측 + 수치예보 + 기상 + 건강 프로필 교차 |
| 창의성 | "지역 숫자"에서 "내 몸 기준 위험 스코어"로 전환 |
| 실현 가능성 | 공공 API 조합, 앱 구현 가능 |

| 논리적 타당성 | 흡입량 = 농도 × 호흡량 × 시간, 과학적 계산 근거 |

## 아이디어 #3 — 우리 집 전기요금 AI 코치 ⭐ 메인 아이디어

**미션**: 에너지효율 (수요관리)
**핵심 키워드**: AMI LP데이터, NILM, 누진제 경보, TOU 최적화, 탄소중립포인트

---

### 한 줄 요약

> 2024년 전국 2,005만 가구에 스마트미터 보급이 완료됐지만, 15분 단위 전력 데이터를 활용한 개인화 서비스는 존재하지 않는다. 이 데이터를 AI로 분석해 "어떤 가전이 문제인지", "누진 구간 언제 넘는지", "요금 얼마나 줄일 수 있는지"를 알려주는 서비스.

---

### 문제 정의

**인프라는 완성됐다. 활용 서비스가 없다.**

한전이 2024년에 AMI(스마트미터) 전국 2,005만 호 보급을 완료했다. 15분 단위 전력 사용량(LP 데이터)이 전국 모든 가구에서 이미 측정되고 있다.

그런데 My한전 앱이 보여주는 건 "이번 달 총 사용량 342kWh" 숫자 하나뿐이다.

- 어떤 가전이 얼마나 쓰는지 모른다
- 이 추세면 누진 몇 단계 진입하는지 모른다
- 어떤 행동을 바꾸면 얼마나 줄일 수 있는지 모른다
- 언제 가전을 돌리는 게 요금이 싼지 모른다

전기요금은 고물가 시대 전국민 관심사인데, 아무도 "어떻게 하면 되는지"를 알려주지 않는다.

---

### 핵심 기술: NILM

**Non-Intrusive Load Monitoring** — 스마트미터 총 사용량 데이터만으로 가전별 소비 패턴을 AI로 역추정하는 기술.

```
[입력] 15분 단위 총 전력 파형

예시 파형:
  00:00~06:00  0.12kW (냉장고 기저 전력)
  07:30~08:00  +0.9kW (세탁기 on — 특유의 모터 스타트 패턴)
  14:00~23:00  +1.8kW (에어컨 on — 주기적 압축기 ON/OFF 패턴)
  23:10~23:55  +0.08kW (대기전력)

        ↓ NILM AI 패턴 분리

[출력]
  냉장고:    월 38kWh (압축기 주기 이상 — 점검 필요 ⚠️)
  에어컨:    월 189kWh (전체 사용량의 47%)
  세탁기:    월 44kWh (화·목·토 오후 3시 패턴)
  TV/대기전력: 월 36kWh (상시 켜있는 구간 감지)
```

별도 스마트플러그 0원. 앱 설치만으로 가전별 분석 가능.
국내 공공 서비스로 제공된 전례 없음 → 독창성 확보.

---

### 서비스 구조 (전체 플로우)

```
[연결] 한전 My데이터 동의 → LP 데이터(15분) 자동 수집

        ↓

[NILM AI 분석]
  총 전력 파형 → 가전별 소비 패턴 역추정
  이상 패턴 감지 (냉장고 소비 급증, 보일러 이상 등)

        ↓

[요금 시뮬레이션]
  현재 누진 구간 위치 실시간 계산
  남은 기간 이 추세면 월말 예상 요금
  TOU 요금제 기준 시간대별 단가 적용

        ↓

[개인화 행동 가이드 — 4가지 기능]

  ① 누진 경보 (가장 핵심)
     "22일에 3단계 진입 예정.
      남은 8일 에어컨 하루 2시간 줄이면 2단계 유지.
      예상 절약: 31,200원"

  ② 최적 시간 가이드
     "세탁기를 오후 3시 → 밤 11시로 이동하면
      경부하 단가 적용 → 월 5,800원 절약"

  ③ 이상 감지 알림
     "냉장고 소비 지난주 대비 40% 증가.
      문이 제대로 닫혔는지 확인하세요."

  ④ 탄소중립포인트 연동
     "이번 달 절약: 38kWh → 탄소 17.3kg 감축
      → 탄소중립포인트 865점 적립 (현금 교환 가능)"
```

### 사용자 시나리오 7개

| 사용자 | 상황 | 서비스 응답 |
|--------|------|------------|
| 주부 | "이번 달 왜 이렇게 나왔지?" | "에어컨 지난달 대비 +189kWh. 17~22일 폭염 구간 원인. 26도 설정 유지 시 2.3만원 절약 가능했음" |
| 1인 가구 | "3단계 진입할 것 같아" | "현재 추세라면 29일 3단계 진입 확정. 지금 에어컨 온도 1도 올리고 세탁은 자정 이후 → 1만8천원 절감" |
| 신혼부부 | "어떤 가전이 제일 많이 먹어?" | "에어컨 47% / 냉장고 21% / 세탁기 11% / 대기전력 9%. 에어컨 패턴 바꾸는 것만으로 효과 최대" |

### 정책 연결

- **에너지 수요관리**: 정부가 피크 수요 감소를 위해 TOU 요금제 확대 중 → 이 서비스가 실질적 수요 분산 실현
- **2024 AMI 전국 보급 완료**: 인프라는 있는데 활용 서비스가 없는 정책 공백 → 직접 해소
- **탄소중립포인트**: 기존 환경부 제도와 연계 → 신규 법제화 불필요, 즉시 연동 가능
- **에너지 취약계층 보호**: 에너지 바우처 사용자 특화 기능 → 복지 정책 연계

### 왜 지금인가

1. **2024 AMI 전국 보급 완료** — 데이터 인프라가 이제야 준비됐다
2. **고물가 시대 전기요금 관심 폭발** — 누구나 절약하고 싶다는 동기 명확
3. **한국 NILM 공공 서비스 전무** — 독창성 확보
4. **에너지 수요 급증 (AI 데이터센터)** — 정부가 수요 관리 절실히 필요한 시점

### 한계 및 보완 사항

- NILM 정확도: 가전 종류가 많거나 동시 가동 시 분리 정확도 저하 → 초기엔 주요 대형 가전 위주로 제한
- 한전 My데이터 API: 사용자 동의 필수 → 온보딩 허들 존재, UX 최소화 필요
- 건물 단열 수준 차이: 동일 사용량이라도 집마다 냉난방 효율 다름 → 주택 정보 추가 연동으로 보완
