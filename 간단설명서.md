# 청정 시간 (Green Hour)

> 5분 안에 전체 그림을 잡기 위한 문서.
> 자세한 근거·출처는 `청정시간_심층조사.md`, 공모 양식은 `청정시간_아이디어_기획서.md` 참조.

---

## 1. 한 줄 요약

> **"내가 새벽 2시에 세탁기 돌리면 한국 전체 탄소 배출이 줄어든다.<br/>AI가 그 시간을 알려주고, 자동으로 보상까지 챙겨준다."**

---

## 2. 왜 이게 문제인가 (3줄)

1. **재생에너지는 시간 따라 들쭉날쭉** 공급된다 (낮에 태양광 폭주, 새벽에 풍력 폭주).
2. **가전은 아무 시간이나** 돌아간다 → 공급-수요가 안 맞으니 결국 화석연료가 메운다.
3. **시민이 시간만 옮겨주면** 재생E가 살고 탄소가 줄어드는데, **알려주는 도구가 한국에 없다.**

---

## 3. 우리가 만드는 것

```
가정의 전기 사용 데이터 (한전 AMI)
              +
시간대별 발전 믹스 (전력거래소 EPSIS)
              +
시간대별 탄소 배출계수 (환경부)
              ↓
        AI가 융합·분석
              ↓
┌──────────────────────────────────────┐
│  "지금 세탁기 돌리세요. 풍력 남아요."  │
│  "오후 7시 피크니까 식기세척기 미루세요." │
└──────────────────────────────────────┘
              ↓
   사용자가 시간 이동 → 자동으로 보상 적립
```

---

## 4. 핵심 한 방 — 충격적인 숫자 하나

| 제도 | 참여 가구 | 보상 단가 |
|---|---|---|
| 한전 에너지캐시백 (절약형) | **166만 호** | 30~100원/kWh |
| 국민DR (시간 분산형) | **7,430호** | **1,300원/kWh** |

**보상이 13~43배 더 높은데 참여율은 1/200.**
이유? **알기 어렵고, 참여하기 어렵고, 효과를 모른다.**

→ **청정 시간이 정확히 그 빈자리를 메운다.**

---

## 5. 다른 서비스와 뭐가 다른가

| | 기존 서비스 | **청정 시간** |
|---|---|---|
| 메시지 | "덜 써라" | **"다른 시간에 써라"** |
| 사용자 부담 | 더위 참기, 덜 쓰기 | **편의 그대로, 시간만 이동** |
| 알림 | 사후 조회 / 발령 시점만 | **24시간 사전 + 실시간** |
| 보상 | 수동 신청·정산 | **4개 제도 자동 통합 정산** |
| 자동화 | ❌ | **스마트 가전 자동 시간 이동** |

> 핵심 한 줄 — **"덜 써라"는 부담이 크지만, "다른 시간에 써라"는 부담이 작다.**

---

## 6. 작동 방식 — 사용자 여정

```
[Step 1] 가입 (5분)
   한전 마이데이터 1회 인증 + 가정 정보 입력

[Step 2] 학습 (1주일)
   AI가 우리 집 패턴 학습
   NILM 기술로 "어떤 가전을 언제 쓰는지" 분리

[Step 3] 알림 받기
   매일 아침 : "오늘의 청정 시간 TOP 3"
   실시간    : "지금 풍력 폭주 → 세탁기 권장"

[Step 4] 행동 + 자동 보상
   알림 따라 시간 이동 → 4개 제도 자동 적립
   스마트 가전이면 예약 자동화로 시작 시간 자동 설정
   EV·온수기·로봇청소기는 완전 자동

[Step 5] 가시화
   "이번 주 절감 = 소나무 2.1그루 1년치, 7,740원 적립"
```

## 7. 4대 핵심 기능

| # | 기능 | 한 줄 |
|---|---|---|
| ① | **청정 시간 알림** | 24시간 미리 + 실시간 푸시로 "지금이 깨끗한 시간"임을 알려줌 |
| ② | **스마트 예약** | EV·온수기·로봇청소기는 완전 자동 / 세탁기·식기세척기는 빨래·그릇 세팅 후 시작 시간 예약을 앱이 자동 설정 |
| ③ | **국민DR 자동 참여** | 발령 시 1탭 참여 → AMI로 자동 측정 → 다음달 요금 차감 |
| ④ | **가시화 + 게임화** | 주간 리포트, 지역 순위, 월간 챌린지로 지속 참여 유도 |

### 7-1. 자동화 수준 — 가전마다 다름 (중요)

| 단계 | 대상 | 사용자 개입 |
|---|---|---|
| 🔔 **알림형** (기본) | 모든 일반 가전 | 알림 보고 사용자가 직접 가전 조작 |
| ⏰ **예약 자동화** | 스마트 세탁기·식기세척기·건조기 (SmartThings·ThinQ API) | 빨래·그릇·세제 미리 세팅 → 앱이 시작 시간 자동 예약 |
| 🤖 **완전 자동** | EV 충전, 온수기, 로봇청소기, ESS·태양광 인버터 | 0 (케이블 꽂기 등 표준 기능 활용) |

> **한국 가정 90%는 알림형이 메인 채널.** 스마트 가전·EV 보유 가정 확대에 따라 자동화 비중이 점진 증가하는 구조.
> "세탁기가 알아서 돌아간다"가 아니라 **"빨래만 넣어두면 시작 시간을 앱이 정해준다"** 가 정확한 표현.

## 8. 통합되는 4개 기존 제도 (신규 0)

```
한전 에너지캐시백  ─┐
국민DR / 에너지쉼표 ─┼─→  청정 시간 앱 하나로
탄소중립포인트     ─┤      자동 신청 + 자동 정산
에너지바우처       ─┘
```

> **신규 제도 신설 0개. 기존 4개를 묶기만 해도 시민 부담이 사라진다.**

## 9. 사용자 시나리오 3컷

#### 케이스 A — 30대 맞벌이 (3인, 30평, 스마트 가전 보유)
- 퇴근 후 빨래·그릇만 넣어두면 → 앱이 새벽 2시 시작을 자동 예약
- **연 ~70만원 절약, CO₂ -180kg**

#### 케이스 B — EV 보유 4인 가구
- 새벽 풍력 강풍대에 EV 충전 자동 예약
- **연 18만원 절감 + CO₂ -240kg**

#### 케이스 C — 에너지바우처 수급 가정
- 자동 옵트인 → 잔액 알림 + 캐시백 자동 적립
- **디지털 격차 없이 에너지 복지 자동화**

## 12. 데이터·기술 스택 (한 눈에)

```
[데이터 6종]
  ① 한전 AMI LP (15분 단위)        ← 국가중점데이터
  ② EPSIS 발전 믹스 (실시간+예측)   ← 국가중점데이터
  ③ 시간대별 탄소 배출계수
  ④ 가전 효율등급 DB                ← 국가중점데이터
  ⑤ 기상청 동네예보
  ⑥ 탄소중립포인트                  ← 국가중점데이터
                                    
                                    → 가점 4종 확보

[AI]
  • NILMFormer (KDD 2025 SOTA Transformer NILM) — 가전별 사용 분리
  • LSTM/Transformer — 시간별 탄소강도 24h 예측
  • 정수계획법 — 청정 시간 매칭 최적화

[서빙]
  • FastAPI + ONNX Runtime
  • React Native (앱) + Next.js (웹)
  • Firebase Cloud Messaging 푸시

[프라이버시]
  • 마이데이터 표준 인증
  • 엣지 추론 옵션 (단말에서 NILM 실행)
  • 차등 프라이버시 + K-익명성 ≥5
```

## 13. 해외 사례 — "이미 작동한다"는 증거

| 나라 | 서비스 | 검증된 것 |
|---|---|---|
| 🇬🇧 영국 | Octopus Agile | **시민이 시간 이동을 한다** (가구당 연 30~50% 절감) |
| 🇺🇸 미국 | OhmConnect | **가정 DR이 도매시장 자원이 된다** (200,000 가구, 1GWh 감축) |
| 🇫🇷 프랑스 | EDF + NILMFormer | **AI 가전 분석이 상용 서비스로 작동한다** (수백만 가구) |

→ **한국은 인프라·법제·데이터 다 있는데, 시민용 통합 서비스만 0.**

## 14. 발표용 한 마디 정리

| 시점 | 메시지 |
|---|---|
| **Why** | "AI 시대에 탄소중립 지키려면, 발전소 더 짓는 게 아니라 **시민의 시간을 바꿔야** 한다." |

## 15. 우리 팀이 답해야 하는 질문 5개

심사위원 / 다른 사람이 물을 때 막히지 말 것.

| 질문 | 답 |
|---|---|
| **"한전이 직접 하면 되지 않나?"** | 한전은 사업자 — 매출 줄이는 절약 코칭에 동기 약함. 우리는 공익 매개자. 영국 Octopus도 발전사 아닌 별도 서비스. |
| **"NILM 정확도 충분한가?"** | 초기엔 대형 가전 위주(에어컨·세탁기·EV·식기세척기). NILMFormer 기반 + 한국 가정 100~500호 시범으로 미세조정. 1분 LP 개방 정책 제언 병행. |
| **"개인정보 우려는?"** | 마이데이터 표준 + 엣지 추론 옵션(단말 실행) + 차등 프라이버시 + K-익명성 5 이상. |
| **"알림이 너무 많지 않나?"** | 일 1~2회 제한 + 자동 모드 우선. 사용자가 알림 없이 자동 이동만 받을 수도. |
| **"사용자가 실제 따라올까?"** | 영국 Octopus·미국 OhmConnect가 이미 입증. 한국은 캐시백 166만 호 참여 의지 확인됨. |
