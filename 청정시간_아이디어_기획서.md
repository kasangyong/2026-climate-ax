## 1. 제안 개요

### 1-1. 구상 및 개발 배경

#### 핵심 인사이트 — 같은 1kWh라도 언제 쓰느냐가 다르다

**같은 1kWh의 전기라도, 사용 시간대에 따라 탄소 배출이 최대 2.7배 차이난다.**

전력망에서 전기를 생산하는 방식은 시간대와 날씨에 따라 실시간으로 달라진다. 낮 12~14시에는 태양광이 정점에 달해 재생에너지 비중이 높아지고, 새벽 1~4시에는 서해안 풍력이 활발하다. 반면 저녁 6~9시에는 가정·상업 수요가 폭증하면서 LNG 화력발전소가 추가로 가동된다. 이 차이가 시간대별 탄소강도(gCO₂/kWh)를 크게 벌린다.

```
【같은 1kWh, 다른 탄소 배출 — 시간대별 탄소강도 개념】

  오후 7시 (저녁 피크, LNG 풀가동)
    → 탄소강도 약 520~550 gCO₂/kWh   ████████████████████████▌

  새벽 1~4시 (풍력 강풍대)
    → 탄소강도 약 280~320 gCO₂/kWh   ███████████

  오전 11시~오후 1시 (태양광 정점)
    → 탄소강도 약 200~250 gCO₂/kWh   ████████▌

  시간만 바꿔도 같은 세탁기 1회 돌리는 탄소가 최대 2.7배 차이.
  (탄소강도 산출 : EPSIS 발전 믹스 × 온실가스종합정보센터 배출계수 가중 평균)
```

그런데 한국 시민은 이 사실을 모르고, 알려주는 서비스가 없다. 세탁기·식기세척기·전기차 충전처럼 **사용 시간을 조정할 수 있는 가전**이 가정에 충분히 있는데도, 언제 쓰는 것이 친환경적인지 알려주는 인터페이스가 한국에는 존재하지 않는다.

---

#### 왜 이 불일치가 심화되고 있는가 — 두 흐름의 충돌

```
[흐름 1] AI 시대 전력 수요 폭증
  - 한국 데이터센터 전력 수요 : 2025년 4,461MW → 2028년 6,175MW (연 +11%)
  - 2038년 예측 : 2025년 대비 약 6배
  - 신규 전력 신청 67%가 수도권 집중
  → AI·클라우드 시대에 24시간 일정한 대량 수요가 폭증

[흐름 2] 재생에너지 비중 급증과 변동성 확대
  - 2025.4 화석연료 발전 비중 49.5% (사상 첫 50% 미만)
  - 태양광 비중 9.2% 사상 최고
  - 2040년 출력제한량 예상 7.8~23.4 TWh (서울시 1년 사용량의 1/3)
  → 재생E는 날씨·시간에 따라 급변, 남을 때 버리고 부족할 때 화석연료로 메움
```

**충돌의 본질** : AI 데이터센터(24시간 일정한 대량 수요)가 폭증하는 동시에, 재생E는 시간·날씨별로 급변한다. 이 불일치를 LNG 화력발전이 메우는 역설이 발생하며, 탄소중립 목표가 뒤흔들린다.

```
【재생E 공급 vs 가정 수요 불일치 — 하루 패턴 개념도 (덕 커브)】

시각         │ 00 │ 02 │ 04 │ 06 │ 08 │ 10 │ 12 │ 14 │ 16 │ 18 │ 20 │ 22 │
-------------|----|----|----|----|----|----|----|----|----|----|----|----|
재생E 발전량 │ ▒▒ │ ▒▒ │ ▒▒ │ ▒▒ │▒▒▒│▓▓▓│▓▓▓│▓▓▓│▒▒▒│ ▒▒ │ ▒▒ │ ▒▒ │
가정 수요    │ ▒▒ │ ▒  │ ▒  │ ▒▒ │▒▒▒│▒▒ │ ▒▒ │▒▒ │▒▒▒│▓▓▓│▓▓▓│▒▒▒│
탄소강도     │보통│낮음│낮음│보통│보통│낮음│최저│낮음│보통│높음│최고│보통│

▓ = 높음  ▒ = 중간
→ 재생E가 남는 낮에 가정이 가전을 쓰면 탄소가 줄고,
  저녁 피크에 쓰면 LNG 발전소를 추가 가동시킨다.
```

**해법은 두 가지뿐이다.**
① 공급 측 : 발전소·ESS 추가 → 수조원의 비용과 수년의 시간이 필요
② 수요 측 : 가정의 전력 사용 **시간을 분산** → 거의 무료, 즉시 가능

그러나 **시민에게 "지금이 청정 시간"임을 알려주는 도구가 한국에 없다.** 영국 Octopus Agile, 미국 OhmConnect는 이미 수십만 가구를 대상으로 작동하고 있는데, 한국은 인프라·법제·데이터가 모두 갖춰졌음에도 시민용 통합 서비스가 전무하다.

### 1-2. 결정적 정책 공백 — 인터페이스 문제

**보상 단가가 13~43배 높은 제도가 있는데 참여율은 1/200이다. 이것은 시민의 의지 문제가 아니라 인터페이스 문제다.**

| 제도 | 참여 가구 | 보상 단가 | 핵심 한계 |
|---|---|---|---|
| 한전 에너지캐시백 (월 절약형) | **166만 호** | 30~100원/kWh | "덜 써라" — 불편함이 전제 |
| 국민DR / 에너지쉼표 (시간 분산형) | **7,430호** (0.45%) | **약 1,300원/kWh** | 신청 어렵고, 발령 통보만, 정산 수동 |

```
【DR 제도 참여 격차 시각화】

AMI 보급 가구    ████████████████████████████████████████  2,005만 호
                 (계량기 이미 설치 완료 — 인프라 ✅)

에너지캐시백     █████████████                              166만 호
                 (절약형 DR, 보상 30~100원/kWh)

국민DR           ▌                                           7,430호
                 (시간이동형 DR, 보상 약 1,300원/kWh)

보상이 13~43배 높은데 참여율이 1/200인 이유 3가지 :
  ① 알기 어렵다    — DR 제도를 아는 시민이 극소수
  ② 참여하기 어렵다 — 개별 신청, 복잡한 절차
  ③ 효과를 모른다  — 내가 얼마를 절감했는지 피드백 없음
```

> **청정 시간은 이 세 가지 장벽을 동시에 제거한다.**
> 알기 쉽게(24시간 사전 알림) + 참여하기 쉽게(1탭 자동 참여) + 효과를 알기 쉽게(즉시 보상 가시화).

### 2-3. AI 기술 — 3개 모델 상세

#### ① NILM — 비침입 부하 모니터링 (가전별 사용 패턴 분리)

**NILM(Non-Intrusive Load Monitoring)** 이란, 가정의 **총 전력 사용량 데이터 하나**로 개별 가전의 사용 패턴을 분리해내는 AI 기술이다. 별도 스마트 플러그나 추가 센서 없이, 이미 설치된 AMI 계량기 데이터만으로 "지금 에어컨이 켜져 있고 세탁기는 꺼져 있다"를 파악한다.

**작동 예시** : 오후 3시 가정 총 전력 사용량이 2.4kW라면, NILM이 "에어컨 1.2kW + 냉장고 0.15kW + TV 0.08kW + 대기전력 0.12kW…"로 분리하고, "세탁기는 꺼져 있으니 청정 시간(오후 4~6시)에 사용 권장"이라는 맞춤 알림을 생성한다.

| 기술 | 내용 |
|---|---|
| **NILMFormer** | 2025년 KDD SOTA Transformer 기반 NILM 모델. EDF 프랑스가 수백만 가구 서비스 백본으로 채택. 오픈소스 공개, 즉시 활용 가능. |
| **사전학습 데이터** | REDD(미국), UK-DALE(영국), ECO(스위스) — 초당~수초 단위 고해상도 공개 데이터셋 |
| **한국 적용 전략** | 15분 해상도 AMI 데이터 → 초기 MVP에서는 에어컨·전기차·전기보일러 등 전력 규모가 큰 대형 가전 위주로 분리. 시범 가구 100~500호에서 라벨링 데이터 수집 후 Active Learning으로 미세조정(fine-tuning). |
| **데이터 해상도 한계** | 학습 데이터(1~6초) vs 한국 AMI(15분, 150배 차이). 초기엔 대형 가전 온/오프 수준 식별. AMI 1분 해상도 개방 후 정밀도 점진 향상. |

#### ② 탄소강도 예측 — 시간별 gCO₂/kWh 산출

**탄소강도(gCO₂/kWh)** 는 전력 1kWh를 생산할 때 나오는 CO₂ 그램 수다. 발전원마다 배출계수가 다르기 때문에, 그 시간의 발전 믹스를 알면 탄소강도를 직접 계산할 수 있다.

```
탄소강도 계산 공식 :
  gCO₂/kWh = Σ (발전원별 발전 비중 × 해당 발전원 배출계수)

예시 (오전 12시, 태양광 많은 날) :
  태양광 35% × 0g  + 풍력 10% × 0g  + LNG 40% × 490g + 석탄 15% × 820g
  = 0 + 0 + 196 + 123 = 319 gCO₂/kWh  ← 낮고 깨끗한 시간

예시 (오후 8시, 저녁 피크) :
  태양광 0% × 0g  + 풍력 5% × 0g  + LNG 70% × 490g + 석탄 25% × 820g
  = 0 + 0 + 343 + 205 = 548 gCO₂/kWh  ← 높고 탄소 집약적
  
  → 동일한 세탁기를 오전 12시에 돌리면 오후 8시 대비 탄소 -42%
```

| 기술 | 내용 |
|---|---|
| **실시간 산출** | EPSIS API 5~15분 발전 믹스 × 온실가스종합정보센터 배출계수 → 즉시 산출 |
| **24시간 예측** | LSTM·Transformer 시계열 모델 + 기상청 일사량·풍속 예보 보조 입력 → 내일 탄소강도 24시간 예측 |
| **글로벌 비교** | 동일 방법론을 WattTime(미국, 5분 단위)·Electricity Maps(190개국)가 이미 상용화. 한국 데이터로 자체 산출 가능 |

#### ③ 청정 시간 매칭 최적화 — 정수계획법 + Pareto 최적해

사용자마다 "허용 시간 창(예: 퇴근 후~내일 아침)"이 다르고, 가전별 가동 시간·제약도 다르다. 단순히 탄소강도 최저 시간을 배정하면 모든 가전이 같은 시간에 몰려 계통에 부담이 된다. 정수계획법으로 다음 세 목표를 동시에 최적화한다.

```
최적화 목표 (3개 동시) :
  ① 탄소강도 최소화  — 가전 가동 시간의 평균 gCO₂/kWh
  ② 요금 최소화      — 시간대별 전기요금 × 가동 전력량
  ③ 사용자 편의 보장 — "희망 종료 시간 이전 완료" 조건 충족

출력 예시 :
  세탁기   : 새벽 02:00 시작 (탄소강도 305 gCO₂/kWh)
  식기세척기: 새벽 03:00 시작 (탄소강도 290 gCO₂/kWh)
  EV 충전  : 새벽 01:30 시작 (풍력 강풍대, 탄소강도 280 gCO₂/kWh)
  → Pareto 최적해 : 탄소·비용 모두 최소화, 편의 조건 충족, 부하 분산
```

### 3-2. 청정 시간의 6가지 차별점 (심화)

#### ① 행동 인식 전환 — "덜 써라"에서 "다른 시간에 써라"로

기존 에너지 절약 캠페인의 핵심은 "소비를 줄여라"다. 이는 더위를 참거나 세탁 횟수를 줄이는 실질적 불편을 요구한다. 반면 청정 시간의 메시지는 **"전기를 쓰되, 더 깨끗한 시간에 써라"** 다.

- **사용자 부담** : 절약(희생) vs 시간 이동(편의 그대로)
- **행동 경제학적 근거** : 현상 유지 편향(status quo bias) — 이미 하던 행동(세탁)의 시간만 옮기는 것은 행동 변화 비용이 극히 낮다
- **영국 Octopus Agile 사례** : 시간대별 동적 요금으로 시민이 자발적으로 가전 시간을 이동, 가구당 연 30~50% 절감. "덜 써라" 없이 달성.

#### ② 한국 최초 가정용 Carbon-Aware Computing

Carbon-Aware Computing은 컴퓨팅 작업을 탄소강도가 낮은 시간대·지역으로 이동시키는 방법론이다. Microsoft Azure와 Google Cloud는 자사 데이터센터의 배치 작업(모델 학습, 백업 등)에 이미 적용 중이다. **청정 시간은 이 개념을 가정의 가전으로 이식하는 한국 최초 시도다.**

- **Microsoft Azure** : Carbon-Aware SDK 오픈소스 공개(2022), 탄소강도 낮은 지역·시간대로 컴퓨팅 부하 이동
- **Google Cloud** : Carbon-Free Energy 시간별 매칭, 2030년 24/7 탄소제로 컴퓨팅 목표
- **청정 시간** : 동일 개념을 세탁기·식기세척기·EV 충전에 적용 — 기업 기술의 가정 민주화

#### ③ NILM 기반 가전별 개인 맞춤 분석 — 한국 최초 상용화

현재 한국에서 가정 전력 데이터를 가전 단위로 분리하는 상용 서비스는 존재하지 않는다. 한전의 My한전 앱은 총 사용량 조회만 제공한다. 청정 시간은 NILMFormer(KDD 2025 SOTA)를 한국 가정 데이터로 미세조정해 "어떤 가전을 언제 쓰는지"를 파악하고, 그 가전의 다음 사용 예정 시간과 청정 시간을 매칭해 **가구 맞춤 알림**을 생성한다.

- **EDF(프랑스) 선례** : NILMFormer를 수백만 가구 에너지 서비스에 적용 — 상용화 가능성 검증
- **한국 MVP 전략** : 초기 에어컨·전기차·전기보일러 대형 가전 위주 → AMI 1분 해상도 개방 후 세탁기·식기세척기 등 정밀화

#### ④ 4개 기존 제도 자동 통합 — 신규 제도 신설 0개

현재 시민이 4개 에너지 보상 제도를 모두 활용하려면 각 기관에 별도 가입, 개별 신청, 개별 정산이 필요하다.

```
현재 (수동·분산) :
  ① 한전 에너지캐시백  → 한전 앱 별도 신청 → 월별 수동 확인
  ② 국민DR / 에너지쉼표 → 전력거래소 별도 가입 → 발령마다 개별 참여
  ③ 탄소중립포인트     → 환경부 별도 사이트 가입 → 실적 별도 조회
  ④ 에너지바우처       → 에너지공단 별도 신청 → 잔액 별도 확인

청정 시간 (통합·자동) :
  최초 1회 한전 마이데이터 인증 → 4개 제도 자동 신청
  → 절감 실적 자동 집계 → 자동 적립 → 통합 대시보드 1개
  → 사용자가 추가로 해야 할 일 : 0개
```

신규 제도나 예산 신설도 없다. 기존 인프라와 제도를 연결하는 인터페이스만 새로 만든다.

#### ⑤ 24시간 사전 청정 시간 공시 — 계획적 생활 지원

기존 국민DR은 발령이 뜨면 그 시점에 통보한다. 시민이 갑작스러운 발령에 대응하기 어렵고, 세탁기나 식기세척기 사용 계획을 미리 바꾸기도 어렵다. 청정 시간은 **매일 아침 다음 24시간의 청정 시간 예측을 공시**한다.

- 매일 오전 7시 : "오늘 청정 시간 TOP 3 — 오전 11시, 오후 2시, 새벽 1시"
- 사용자가 출근 전에 세탁기 빨래를 넣고 "자동 예약" 설정 → 앱이 알아서 청정 시간에 시작
- 야간 EV 충전 : 전날 밤 케이블 꽂으면 새벽 풍력 강풍대에 자동 완충

#### ⑥ 자동 모드 — 알림 피로 없이 탄소 감축

알림 기반 서비스의 공통 실패 원인은 알림 피로(notification fatigue)다. 청정 시간은 알림에 응답하지 않아도 작동하는 **자동 모드**를 제공한다. 스마트 가전·EV를 보유한 사용자는 자동 모드를 켜두면 알림 없이 시간 이동이 자동 실행된다.

- SmartThings·ThinQ API 연동 → 세탁기·건조기·식기세척기 시작 시간 자동 예약
- EV 충전기 API 연동 → 케이블 꽂으면 최적 시간에 자동 충전 시작
- 전기 온수기·로봇청소기 → 스케줄 자동 이동

### 3-4. 해외 검증 — "이미 작동한다"는 증거

| 사례 | 규모 | 검증 내용 | 한국 적용 포인트 |
|---|---|---|---|
| **Octopus Agile (영국)** | 수십만 가구 | 30분 단위 동적 요금 → 시민이 자발적 시간 이동, 가구당 연 30~50% 절감 | 시민이 인터페이스만 있으면 시간을 이동한다는 것을 입증 |
| **OhmConnect (미국)** | 약 200,000 가구 | 캘리포니아·텍사스, 한 주 1GWh 감축, 가정 DR을 도매시장 자원화 | 가정 DR이 계통 자원으로 작동 가능함을 입증 |
| **EDF / NILMFormer (프랑스)** | 수백만 가구 | KDD 2025 SOTA 모델을 실서비스 백본으로 채택 | AI 가전 분석이 상용화 수준임을 입증 |
| **WattTime / Electricity Maps** | 190개국 | 5분 단위 탄소강도 API 표준화, Microsoft·Google Cloud 채택 | 탄소강도 실시간 산출 방법론 이미 검증 |

> **결론** : 한국은 인프라(AMI 2,005만 호)·법제(분산E 특별법 2024.6 시행)·데이터(EPSIS·배출계수 개방)가 모두 갖춰져 있다. 없는 것은 **시민과 제도를 연결하는 통합 인터페이스뿐**이다.

### 4-2. 4대 핵심 기능 상세

#### ① 청정 시간 알림 (Green Hour Push)

```
[매일 새벽 5시 배치 실행 — 알림 생성 로직 개요]

for each user:
    user_profile    = NILM_result(user.AMI_LP_7days)      // 가전별 패턴
    flexible_load   = identify_flexible(user_profile)      // 시간 이동 가능 가전
    carbon_forecast = predict_24h(EPSIS_realtime, weather) // 탄소강도 예측
    peak_warning    = detect_peak_hours(carbon_forecast)   // 피크 시간 탐지
    optimal_slots   = match(flexible_load, carbon_forecast) // 최적 시간 매칭

    if user.smart_mode == ON:
        auto_schedule(flexible_load, optimal_slots)        // 자동 예약
    else:
        push_notification(optimal_slots, peak_warning)     // 알림 발송
```

**알림 유형 3가지 :**

| 유형 | 발송 시점 | 예시 메시지 |
|---|---|---|
| **일일 청정 시간 TOP 3** | 매일 오전 7시 | "오늘 11~13시 태양광 정점 — 세탁기 시 탄소 -68%" |
| **실시간 기회 알림** | 재생E 급증 시 | "지금 풍력 폭주 → 식기세척기 권장, 비용 -180원" |
| **피크 경고** | 수요 피크 1시간 전 | "오후 6시 LNG 피크 시작 — 큰 가전 30분 미루면 -8%" |

> 알림은 일 최대 2회로 제한. 자동 모드 이용자는 알림 없이 자동 실행만 수신.

#### ② 스마트 예약 (Smart Schedule) — 3단계 자동화

| 단계 | 대상 가전 | 사용자 개입 | 기술 메커니즘 |
|---|---|---|---|
| **알림형** (기본·90%) | 모든 일반 가전 | 알림 보고 직접 조작 | 푸시 알림 → 사용자 행동 |
| **예약 자동화** | 스마트 세탁기·식기세척기·건조기 | 빨래·그릇 미리 세팅 | SmartThings·ThinQ API로 시작 시간 자동 예약 |
| **완전 자동** | EV·전기 온수기·로봇청소기·ESS | 케이블 꽂기 등 표준 행동 | API 연동, 사용자 개입 0 |

**예약 자동화 상세 흐름 (스마트 세탁기 예시) :**

```
[사용자]  퇴근 후 빨래·세제 투입
          앱에서 "자동 예약 ON" + 허용 시간 창 설정 (저녁 6시~내일 오전 9시)

[AI]      해당 시간 창 안에서 탄소강도·요금 최솟값 탐색
          → 결정: 새벽 2시 (탄소강도 310 gCO₂/kWh, 야간 요금 적용)

[API]     Samsung SmartThings 또는 LG ThinQ API에 예약 명령 전송
          → 세탁기 새벽 2시 시작 예약 완료

[완료]    오전 6시 완료 알림
          "세탁 완료 🌿 탄소 -52%(평소 저녁 대비), 비용 -180원, 포인트 +85점"
```

#### ③ 국민DR 자동 참여 (DR Auto-Join)

**현재 국민DR 참여 흐름 (수동, 장벽 3개) :**
```
전력거래소 별도 가입 → 발령 문자 수신 → 사이트 접속 → 수동 참여 신청
→ 익월 요금 청구서 확인 → 수동 정산 요청
```

**청정 시간 DR 참여 흐름 (1탭, 장벽 0개) :**
```
DR 발령 (전력거래소) : "오후 2~5시 감축 시 1,300원/kWh 보상"
       ↓
앱 푸시 : "[국민DR 발령] 오후 2~5시 큰 가전 자제
           예상 적립 : 5,200원 → [참여]"
       ↓
1탭 참여 → AMI가 실제 감축량 자동 측정
       ↓
다음달 전기 요금 자동 차감 (정산 수동 절차 없음)
```

> 현재 7,430호 정체 원인이 개별 신청·수동 관리 부담 → 자동화 시 10만 호 단위 확대 가능.

### 4-4. 페르소나 시나리오 (4종)

#### 케이스 A — 30대 맞벌이 부부 (3인, 30평, 스마트 가전 보유)
- **상황** : 퇴근 오후 7시, 세탁기 빨래 넣고 식기세척기 그릇 세팅
- **행동** : "자동 모드 ON" 터치 1번
- **자동 실행** : AI가 새벽 2시 세탁기 + 새벽 3시 식기세척기 자동 예약 (SmartThings API)
- **효과** : 연 ~70만원 절약, CO₂ -180kg, 아침에 빨래·설거지 완료 상태

#### 케이스 B — EV 보유 4인 아파트
- **상황** : 저녁 귀가 후 충전 케이블 꽂음 (앱 별도 조작 없음)
- **자동 실행** : 새벽 1~4시 풍력 강풍대 감지 → 자동 충전 시작
- **효과** : 연 18만원 절감(심야 요금 + DR 보상), CO₂ -240kg, 아침 만충

#### 케이스 C — 에너지바우처 수급 고령 1인 가구
- **상황** : 디지털 취약계층, 앱 자체 조작 어려움
- **행동** : 가입 시 자동 옵트인 (자녀가 대신 설정 가능)
- **효과** : 바우처 소진 전 자동 알림 + 캐시백 자동 적립, 디지털 격차 없이 에너지 복지 혜택

#### 케이스 D — 아파트 관리사무소 (B2B, 500세대 단지)
- **상황** : 관리사무소가 단지 전체 에너지 관리 플랫폼으로 청정 시간 도입
- **자동 실행** : 지하 공용 충전기 야간 자동 전환 + 엘리베이터 청정 시간 우선 운영
- **효과** : 단지 에너지비 절감 + 입주민 보상 배분 + 녹색 아파트 인증 가점

| EPSIS 발전 믹스 실시간 API | ✅ 개방 완료 | 표준 Open API (epsis.kpx.or.kr) |
| EPSIS 24h 예측 | ⚠️ 비공식 참고 수준 | 기상 보조 예측으로 MVP 보완, 7항 정책 제언 병행 |
| NILMFormer 등 NILM 오픈소스 | ✅ 즉시 활용 가능 | NILMTK, Torch-NILM, NeuralNILM-PyTorch |
| 탄소중립포인트 API | ✅ 연동 가능 | 환경부 cpoint.or.kr |
| 분산에너지 활성화 특별법 | ✅ 2024.6 시행 완료 | **법 개정 불필요** — VPP 법적 근거 확보 |

> **실질 병목** : "API 개방됨"과 "서비스에 바로 사용 가능"은 다르다. 한전 마이데이터 사업자 협약·심사, 전력거래소 정산 연동 협약은 6~12개월 소요될 수 있다. MVP 개발과 병행해 기관 협의를 선제 착수하는 것이 핵심이다.

## 6. 사회 문제 해결

### 6-1. AI 시대 전력 수요 폭증과 탄소중립 위협
- 데이터센터 2038년까지 약 6배 증가, AI 서버는 24시간 일정한 수요 → 재생E 변동성과 정반대
- **해법** : 가정 부문 시간 분산이 가장 빠르고 저렴한 수요 측 완충 수단

### 6-2. 재생에너지 출력제한 (버려지는 전기)
- 2040년 23.4 TWh 출력제한 예상 (서울시 1년 사용량의 1/3)
- **해법** : 청정 시간 알림으로 가정이 잉여 재생E 흡수 → 출력제한 감소

### 6-3. 가정 DR 참여 정책 공백
- 보상 13~43배 높은 국민DR 참여율 0.45% — 인터페이스 문제
- **해법** : 자동 알림·1탭 참여·자동 정산으로 진입 장벽 완전 제거

### 6-4. 에너지 복지 사각지대
- 에너지바우처 수급자의 잔액 소진 관리, 디지털 접근성 어려움
- **해법** : 자동 옵트인 + 잔액 실시간 알림 + 캐시백 자동 적립으로 디지털 격차 해소

### 6-5. 시민의 환경 행동 동기 부족
- 추상적 "탄소중립" 메시지로는 행동 변화 한계
- **해법** : 즉각적인 경제적 보상(연 5~10만원) + 소나무 환산 가시화 + 지역 랭킹 게임화

### 6-6. 가정 부문 에너지 정보 비대칭
- 시간대별 탄소강도 정보가 시민에게 전혀 제공되지 않음
- **해법** : 청정 시간 앱을 통한 시간대별 탄소 배출계수 실시간 공시 + 정부 공시 의무화 정책 제언
